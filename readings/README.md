# Readings — Learner Reference Material

These Markdown files provide the depth you need to study the course **offline**, before or after the live sessions.

The course follows the May 2026 exam structure (7 domains with published weights). The weekly readings are organized around the same 5-Friday progression that maps cleanly onto both the 7-domain breakdown and the higher-level groupings often referenced in official prep paths (Intelligence Platform, Development & Ingestion, Data Processing & Transformations, Productionizing Data Pipelines, and Governance & Quality).

## Purpose

- Slides give the visual overview and key bullets.
- Notebooks give you the hands-on practice.
- These readings give you the **why**, the mental models, decision frameworks, and exam-focused explanations in clear prose you can read anywhere.

Use them to:
- Preview concepts before class (20–30 minutes).
- Review and solidify understanding after the lab.
- Cram targeted topics during your final week before the exam.

## How to Use These Readings

1. **Before class** — Read the "Overview" and the first 2–3 concept sections to activate prior knowledge.
2. **After class / lab** — Read the full file + complete the self-check questions.
3. **Exam week** — Re-read the "Exam Focus" and "Key Mental Models" sections for your weakest domains. Pair with re-doing that week's lab from scratch.

## Relationship to Other Materials

- **Solutions/** — Released *after* you attempt the lab. These are "what you built + patterns + mistakes".
- **Readings/** — Always available. These are the conceptual reference you can study independently.
- **Notebooks/** — The actual code you run.
- **visuals/** — Diagrams referenced in several readings (Data Intelligence Platform Stack, BrewMart Medallion Flow, etc.).

## Tips for Maximum Value

- Read actively: cover the code examples and try to predict what happens before looking.
- Pay special attention to sections labeled **Exam Trap** or **Decision Framework** — these directly address common wrong answers.
- Keep your own "rules wall" notes as you go (photograph the key decision tables).
- Revisit the BrewMart example in each reading — the same dataset threads through the entire course.

These readings were created to close the "no dedicated learner reading material" gap identified in the course audit. They are designed to be used alongside the official Databricks documentation listed at the end of each file.

Happy studying — and good luck on the exam!
